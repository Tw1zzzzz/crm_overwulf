# Исправление проблемы доступа к карточкам игроков

## Проблема
Пользователи со статусом staff не могли загрузить список карточек игроков. В логах сервера отображалась ошибка 403:

```
🔍 [3vqeyhg1] GET /api/player-cards?page=1&limit=50 - начало запроса
[AUTH] Проверка авторизации для GET /api/player-cards?page=1&limit=50
[AUTH] Заголовки: Bearer token присутствует
[AUTH] Токен получен, проверяю...
[AUTH] Токен действителен, ID пользователя: 685908d44edc81107d2a4b1d
[AUTH] Проверка привилегий: ключ из env загружен: true
[AUTH] Доступ запрещен: неверный ключ привилегий или его отсутствие
✅ [3vqeyhg1] GET /?page=1&limit=50 - статус 403 (26ms)
```

## Анализ причины
Маршрут `/api/player-cards` в файле `src/server/routes/playerCards.ts` использовал middleware `hasPrivilegeKey` для ВСЕХ операций, включая просмотр карточек. Это создавало излишние ограничения:

**Исходный код:**
```typescript
// Маршруты доступные только для сотрудников (staff) с ключом привилегий
router.use(protect);
router.use(isStaff);
router.use(hasPrivilegeKey); // <- Применялся ко всем маршрутам
```

Проблема заключалась в том, что ключ привилегий требовался даже для операций просмотра, хотя логично требовать его только для операций редактирования/удаления.

## Решение
Изменена структура маршрутов в `src/server/routes/playerCards.ts`:

### Маршруты для просмотра (только staff)
- `GET /` - получить все карточки игроков
- `GET /:userId` - получить карточку конкретного игрока

### Маршруты для редактирования (требуют hasPrivilegeKey)
- `POST /` - создать карточку игрока
- `PUT /attach-player` - привязать игрока к карточке
- `PUT /:userId/contacts` - обновить контакты
- `PUT /:userId/communication-line` - обновить коммуникативную линию
- `POST /:userId/roadmap` - загрузить roadmap
- `POST /:userId/mindmap` - загрузить mindmap
- `POST /:userId/communication-image` - загрузить изображение
- `DELETE /:userId` - удалить карточку

**Обновленный код:**
```typescript
// Маршруты для просмотра - доступны для всех сотрудников (staff)
router.use(protect);
router.use(isStaff);

// Получить все карточки игроков с пагинацией (только просмотр)
router.get('/', getAllPlayerCards);

// Получить карточку игрока по ID (только просмотр)
router.get('/:userId', getPlayerCard);

// Маршруты для редактирования - требуют ключ привилегий
router.post('/', hasPrivilegeKey, createPlayerCard);
router.put('/attach-player', hasPrivilegeKey, attachPlayerToCard);
// ... остальные операции редактирования
```

## Преимущества нового подхода
1. **Логичное разделение доступа**: просмотр доступен всем staff, редактирование - только с ключом привилегий
2. **Соответствие с другими маршрутами**: например, `/api/users/players` также требует только staff для просмотра
3. **Лучший UX**: пользователи могут видеть карточки игроков без дополнительных настроек
4. **Безопасность**: критичные операции по-прежнему защищены ключом привилегий

## Проверка работоспособности
После изменений необходимо:
1. Перезапустить сервер
2. Войти под учетной записью staff
3. Перейти на вкладку "Карточки игроков"
4. Убедиться, что список загружается без ошибок 403

## Дополнительная проблема Frontend
После исправления API была обнаружена вторая проблема: в диалоге создания карточки показывалось "Загрузка игроков..." потому что диалог открывался синхронно, а загрузка игроков происходила асинхронно.

### Исправление Frontend
Изменена функция `handleOpenCreatePlayerCardDialog` в `src/pages/PlayerCard.tsx`:
- Диалог теперь открывается только после успешной загрузки игроков
- Добавлен индикатор загрузки на кнопку "Добавить карточку"
- Улучшена обработка ошибок загрузки

**Изменения в коде:**
```typescript
const handleOpenCreatePlayerCardDialog = async () => {
  try {
    resetAllPlayerStates();
    
    // Всегда загружаем свежий список игроков перед открытием диалога
    setLoadingPlayers(true);
    const result = await getPlayers();
    
    if (result.data && Array.isArray(result.data)) {
      const normalizedPlayers = result.data
        .filter(p => p && (p._id || p.id) && p.role === "player")
        .map(p => ({
          id: (p as any)._id || p.id,
          name: p.name || "Неизвестно",
          email: p.email || "",
          role: p.role
        }));
      
      setPlayers(normalizedPlayers);
      
      // Открываем диалог только после успешной загрузки игроков
      setShowAddPlayerDialog(true);
    } else {
      toast.error("Не удалось загрузить список игроков");
    }
  } catch (error) {
    toast.error('Не удалось загрузить список игроков');
  } finally {
    setLoadingPlayers(false);
  }
};
```
