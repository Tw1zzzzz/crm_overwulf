# CORRELATION ANALYSIS REFACTORING - Рефакторинг корреляционного анализа

## 🎯 Цель рефакторинга
Полное переосмысление функционала корреляционного анализа с переходом от анализа "до/после отчетов" к мультиметричному dashboard'у с возможностью ввода новых типов данных.

## 📊 Проблемы старой системы
1. **Неправильная концепция**: Анализ причинно-следственной связи между отчетами и настроением
2. **Ограниченные данные**: Только настроение и колесо баланса
3. **Отсутствие ввода данных**: Нет возможности добавлять игровые показатели и экранное время
4. **Плохая визуализация**: Разрозненные данные без единого dashboard'а

## 🏗️ Новая архитектура

### Этап 1: Подготовка архитектуры ✅
- **Новые модели данных**:
  - `ScreenTime.ts` - экранное время (общее время, развлечения, общение, браузер, учеба)
  - `GameStats.ts` - игровые показатели (матчи, K/D, раунды, специальные показатели)
- **API контроллеры**:
  - `screenTimeController.ts` - управление экранным временем
  - `gameStatsController.ts` - управление игровыми показателями
- **Маршруты**:
  - `/api/screen-time` - CRUD операции для экранного времени
  - `/api/game-stats` - CRUD операции для игровых показателей
- **Навигация**: Вынос корреляционного анализа в отдельное меню

### Этап 2: Формы ввода данных ✅
- **ScreenTimeForm**: Форма ввода экранного времени с валидацией (макс. 24 часа)
- **GameStatsForm**: Форма ввода игровых показателей с автоматическими расчетами

### Этап 3: Новая страница корреляций ✅
- **Структура**: Tabs с тремя вкладками
  - Анализ корреляций - мультиметричный график
  - Экранное время - форма ввода данных
  - Игровые показатели - форма ввода данных
- **Визуализация**: Интерактивные графики как в dashboard'е
- **Фильтры**: Временные периоды и выбор метрик

### Этап 4: Выбор игроков и режимов анализа ✅
- **Режимы анализа**:
  - Командная статистика - анализ всех игроков команды
  - Индивидуальная статистика - анализ конкретного игрока
- **Выбор игрока**: Dropdown с загрузкой из API `/api/users/players`
- **Динамические заголовки**: Заголовки графиков меняются в зависимости от режима
- **Валидация**: Проверка выбора игрока в индивидуальном режиме
- **Автоматический сброс**: Сброс данных при смене режима анализа
- **API исправления**: Исправлены endpoints во всех формах:
  - CorrelationAnalysisPage: `/api/auth/players` → `/api/users/players`
  - ScreenTimeForm: `/api/users` → `/api/users/players`
  - GameStatsForm: `/api/users` → `/api/users/players`

## 🔧 Технические детали

### Модели данных
```typescript
// ScreenTime - экранное время
interface IScreenTime {
  userId: ObjectId;
  date: Date;
  totalTime: number;        // Общее экранное время
  entertainment: number;    // Развлечения (игры, видео, фильмы)
  communication: number;    // Общение (соцсети, мессенджеры, звонки)
  browser: number;          // Браузер (серфинг, новости, покупки)
  study: number;            // Учеба (курсы, образовательные материалы, работа)
  calculatedTotal: number;  // Автоматически рассчитывается сумма категорий
}

// GameStats - игровые показатели
interface IGameStats {
  userId: ObjectId;
  date: Date;
  // Общие показатели матчей
  totalMatches: number;     // Общее количество матчей
  wins: number;             // Победы
  losses: number;           // Поражения
  draws: number;            // Ничьи
  winRate: number;          // Win-Rate в % (автоматически рассчитывается)
  
  // K/D статистика
  kills: number;            // Убийства
  deaths: number;           // Смерти
  assists: number;          // Помощи
  kdRatio: number;          // K/D соотношение (автоматически рассчитывается)
  
  // Статистика раундов
  roundsWon: number;        // Выигранные раунды
  roundsLost: number;       // Проигранные раунды
  totalRounds: number;      // Общее количество раундов (автоматически рассчитывается)
  roundWinRate: number;     // Win-Rate раундов в % (автоматически рассчитывается)
  averageRoundsWon: number; // Средние выигранные раунды за матч (автоматически рассчитывается)
  averageRoundsLost: number;// Средние проигранные раунды за матч (автоматически рассчитывается)
  
  // Специальные раунды
  pistolRounds: number;     // Пистолетные раунды
}
```

### API Endpoints
- `GET /api/screen-time` - получение данных экранного времени
- `POST /api/screen-time` - создание/обновление записи (только staff)
- `GET /api/screen-time/stats` - статистика по всем игрокам (только staff)
- `GET /api/game-stats` - получение игровых показателей
- `POST /api/game-stats` - создание/обновление записи (только staff)
- `GET /api/game-stats/overview` - общая статистика (только staff)

### Права доступа
- **Просмотр данных**: Игроки видят только свои данные, staff видит всех
- **Ввод данных**: Только staff может вводить данные через формы
- **Корреляционный анализ**: Только staff имеет доступ к странице
